# -H.O.D.O.R
This Pokédex is built for us—because we always wanted to know everything about Pokémon, but never had a truly cool one—so we created our own to explore, search, and experience them the way we imagined.

##  Features

###  1. Pokémon Data Management

Stores detailed information for each Pokémon, including:

- Name
- Type(s)
- Abilities
- Base stats (HP, Attack, Defense, Speed, etc.)
- Evolution chain

Uses structured formats like JSON or databases and is easily extendable for new Pokémon data.

---

###  2. Search & Filtering System

Search Pokémon by:

- Name
- Type
- ID number

Apply filters:

- Type-based filtering
- Region-based filtering
- Evolution stage

Provides fast and responsive results.

---

###  3. Unique Identification System

Each Pokémon has a unique ID. For example:

- `#001` → Bulbasaur
- `#025` → Pikachu

This enables easy indexing, sorting, and efficient referencing.

---

###  4. Classification & Organization

Organizes Pokémon by:

- Region (Kanto, Johto, etc.)
- Type (Fire, Water, etc.)
- Evolution stage

Improves navigation and overall structure.

---

###  5. Visual Display System

- Displays Pokémon images (sprites/artwork)
- Supports multiple forms (Shiny, Mega, etc.)
- Responsive layouts for better visual recognition

---

###  6. Location / Habitat Information

Shows where Pokémon can be found, including:

- Regions
- Specific habitats (forest, cave, water, etc.)

Useful for gameplay guidance and exploration.

---

###  7. Description / Pokédex Entries

Provides descriptive text for each Pokémon covering:

- Behavior
- Characteristics
- Fun facts

Mimics the classic Pokédex style from the games.

---

###  8. User Interface (UI)

- Clean and intuitive design
- Search bar, Pokémon cards/list view, and detailed info pages
- Fully responsive and mobile-friendly

---

###  9. API Integration

Fetches real-time data from APIs (e.g., [PokéAPI](https://pokeapi.co/)) and handles:

- Data fetching
- Loading states
- Error handling

Keeps the application dynamic and up-to-date.
